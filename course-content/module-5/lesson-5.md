# Lesson 5.5: Translating Design to Cursor Prompts

**Estimated Time**: 25 minutes

---

## Introduction

You've planned your website type, pages, colors, and typography. Now you need to communicate all of this to Cursor in a way that produces the website you envision. This lesson shows you how to write effective Cursor prompts that translate your design decisions into reality.

**What You'll Learn:**
- How to structure effective Cursor prompts
- What information to include
- How to be specific without being overwhelming
- Example prompts for different scenarios
- Common prompt mistakes to avoid
- How to refine prompts for better results

---

## Why Prompt Quality Matters

### Good Prompts = Good Results

**A good prompt:**
- Gives Cursor all the information it needs
- Is clear and specific
- Includes your design decisions
- Produces results that match your vision

**A bad prompt:**
- Vague or unclear
- Missing important information
- Too generic
- Produces results that don't match your vision

**The better your prompt, the better your website!**

---

## The Complete Design Prompt Structure

### Essential Components

**A complete design prompt includes:**
1. **Website type** - What kind of website
2. **Target audience** - Who it's for
3. **Pages needed** - What pages to create
4. **Color palette** - Your chosen colors
5. **Typography** - Your font choices
6. **Style/Theme** - Overall design direction
7. **Specific requirements** - Any special needs

---

## Building Your Prompt Step by Step

### Step 1: Start with the Basics

**Include:**
- Website type
- Target audience
- Main goal

**Example:**
```
I want to build a booking website for a lash extension studio.
Target audience: Women 25-45 interested in beauty and self-care.
Main goal: Allow clients to book appointments online.
```

---

### Step 2: Add Your Pages

**List the pages you need:**
- Be specific about what each page should contain

**Example:**
```
Pages needed:
- Home: Services overview, trust indicators, booking CTA
- Services: Detailed service list with prices and durations
- Booking: Service selection, calendar, booking form
- About: Studio story, stylist experience
- Contact: Location, hours, phone, email
```

---

### Step 3: Include Your Colors

**Provide:**
- Color names and hex codes
- Which colors are for what (primary, secondary, accent)
- Overall color mood

**Example:**
```
Color palette:
- Primary: Soft pink (#F5D7D7)
- Secondary: Cream (#FFF8F0)
- Accent: Gold (#D4AF37)
- Text: Dark brown (#3E2A20)
- Background: White (#FFFFFF)
Style: Soft, feminine, elegant, modern
```

---

### Step 4: Add Typography

**Specify:**
- Heading font
- Body font
- Font sizes if important
- Typography style

**Example:**
```
Typography:
- Headings: Poppins (sans-serif, semi-bold)
- Body: Inter (sans-serif, regular, 16px)
- Style: Clean, modern, highly readable
```

---

### Step 5: Describe the Overall Style

**Include:**
- Design theme (minimal, elegant, modern, etc.)
- Mood/feeling you want
- Any style references

**Example:**
```
Overall style: Minimal, modern, clean, aesthetic
Mood: Soft, feminine, elegant, professional
Design approach: Clean layouts, plenty of white space, 
focus on content and usability
```

## Prompt Refinement Tips

### Start Broad, Then Get Specific

**First prompt (broad):**
```
Build a website for my business.
```

**Refined prompt (specific):**
```
Build a booking website for a lash studio with soft pink and cream colors, 
Poppins headings, Inter body text, minimal modern style, for women 25-45.
Pages: Home, Services, Booking, About, Contact.
```

---

### Use Examples and References

**Good:**
```
Style similar to [website example], but with my color palette 
(soft pink #F5D7D7, cream #FFF8F0, gold accents).
```

**Or:**
```
Clean, minimal style like Apple's website, but warmer and more feminine 
with soft pink tones.
```

---

### Be Specific About Layout

**Good:**
```
Homepage layout:
- Hero section with headline, subheadline, and CTA button
- Services preview section with 3-4 service cards
- Trust indicators section (testimonials or stats)
- Footer with contact info and social links
```

---

### Include Technical Requirements

**If relevant:**
```
- Responsive design (mobile, tablet, desktop)
- Fast loading
- Accessible (WCAG compliant)
- SEO-friendly structure
```

## Common Prompt Mistakes

### 1. Too Vague

**Bad:**
```
Make a website for my business.
```

**Good:**
```
Build a booking website for a lash studio. Target audience: women 25-45. 
Pages: Home, Services, Booking, About, Contact. Colors: soft pink, cream, gold. 
Style: minimal, modern, elegant.
```

---

### 2. Too Many Conflicting Instructions

**Bad:**
```
Make it bold and vibrant but also soft and minimal, professional but playful, 
modern but traditional.
```

**Good:**
```
Style: Minimal, modern, soft colors, elegant. Mood: Warm, inviting, professional.
```

---

### 3. Missing Key Information

**Bad:**
```
Build a website with pink colors.
```

**Good:**
```
Build a booking website. Colors: Soft pink (#F5D7D7) primary, cream (#FFF8F0) 
secondary, gold (#D4AF37) accent. Pages: Home, Services, Booking, Contact.
```

---

### 4. Unrealistic Expectations

**Bad:**
```
Build a complete e-commerce site with payment, inventory, shipping, 
customer accounts, reviews, and AI chatbot in one prompt.
```

**Good:**
```
Build the homepage for my e-commerce site. Include hero section, featured 
products section, and navigation. We'll add other features in separate prompts.
```

---

## Iterative Prompting Strategy

### Build in Stages

**Stage 1: Structure**
```
Create the basic page structure and layout for my homepage.
Include: header, hero section, services preview, footer.
```

**Stage 2: Styling**
```
Apply the color palette (soft pink #F5D7D7, cream #FFF8F0, gold #D4AF37) 
and typography (Poppins headings, Inter body) to the homepage.
```

**Stage 3: Content**
```
Add content to the homepage: compelling headline, service descriptions, 
trust indicators, call-to-action buttons.
```

**Stage 4: Refinement**
```
Refine the homepage: improve spacing, adjust colors, enhance typography, 
ensure mobile responsiveness.
```

## Key Takeaways

1. **Be specific** - Include all your design decisions (colors, fonts, pages, style)
2. **Start with basics** - Website type, audience, pages
3. **Add design details** - Colors, typography, style
4. **Use examples** - Reference styles you like
5. **Build iteratively** - Start with structure, then style, then content
6. **Refine as needed** - Improve prompts based on results
7. **Keep it focused** - One main request per prompt
8. **Include context** - Target audience and goals help Cursor understand
